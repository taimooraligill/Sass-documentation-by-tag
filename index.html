Introduction

Sass (Syntactically Awesome Style Sheets) is a preprocessor scripting language that is interpreted or compiled into Cascading Style Sheets (CSS). SassScript is the scripting language itself. Sass consists of two syntaxes. The original syntax, called "the indented syntax," uses a syntax similar to Haml. It uses indentation to separate code blocks and newline characters to separate rules. The newer syntax, "SCSS," uses block formatting like that of CSS.

  • Enhances CSS: Sass provides advanced features that allow you to write CSS more efficiently.
  • Nesting: You can nest your CSS selectors in a way that follows the same visual hierarchy of your HTML.
  • Variables: Use variables to store CSS values that you reuse throughout your stylesheet.
  • Mixins: Create reusable chunks of CSS code with mixins.
  • Inheritance: Share a set of CSS properties from one selector to another using inheritance.

Installation

Using npm

To install Sass via npm, use the following command:

npm install -g sass

Using RubyGems

To install Sass via RubyGems, use the following command:

gem install sass

Using Homebrew

To install Sass via Homebrew, use the following command:

brew install sass

Using a Package Manager

Sass can also be installed using various package managers, such as Yarn:

yarn global add sass

Basic Syntax

Sass offers two syntaxes: SCSS and the indented syntax.

SCSS Syntax


    $primary-color: #333;

    body {
      color: $primary-color;
    }

Indented Syntax


    $primary-color: #333

    body
      color: $primary-color

Nesting

Sass allows you to nest your CSS selectors in a way that follows the same visual hierarchy of your HTML.


    nav {
        ul {
          margin: 0;
          padding: 0;
          list-style: none;
        }
      
        li { display: inline-block; }
      
        a {
          display: block;
          padding: 6px 12px;
          text-decoration: none;
        }
      }

Variables

Variables make it possible to reduce repetition and keep your code DRY.


    $font-stack: Helvetica, sans-serif;
    $primary-color: #333;
    
    body {
    font: 100% $font-stack;
    color: $primary-color;
    }

Partials & Import

Partials allow you to create small Sass files that contain snippets of CSS that can be included in other Sass files. A partial is created by prefixing the filename with an underscore.


    // _reset.scss
    body {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
    }

    // main.scss
    @import 'reset';

    .container {
    width: 100%;
    }

Mixins

Mixins allow you to create reusable chunks of code.


    @mixin border-radius($radius) {
        -webkit-border-radius: $radius;
           -moz-border-radius: $radius;
            -ms-border-radius: $radius;
                border-radius: $radius;
      }
      
      .box { @include border-radius(10px); }

Inheritance

Inheritance allows you to share a set of CSS properties from one selector to another.

                
    .message {
        border: 1px solid #ccc;
        padding: 10px;
        color: #333;
      }
      
      .success { @extend .message; border-color: green; }
      .error   { @extend .message; border-color: red; }
      .warning  @extend .message; border-color: yellow;

Operators

Sass has several built-in operators for common mathematical operations, including +, -, *, /, and %.

                
    .container {
        width: 100% - 20px;
      }
      
      .sidebar {
        width: 300px / 960px * 100%;
      }

Functions

Sass provides built-in functions for manipulating colors, numbers, strings, lists, and maps.

                
    $color: #4D926F;

    .button {
    color: lighten($color, 20%);
    background-color: darken($color, 20%);
    }

Comments

Sass supports both standard CSS comments (/* comment */) and inline comments (// comment).

                
    /* This is a standard comment */
    body {
    font-family: sans-serif; // This is an inline comment
    }

Compiling SASS

To compile Sass to CSS, you can use the command line tool provided by Sass.

                
    sass input.scss output.css

You can also watch a directory for changes and automatically compile the Sass files to CSS:

                
    sass --watch input.scss:output.css
